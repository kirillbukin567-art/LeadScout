# LeadScout — краткое описание системы (для разработчика)

**Заказчик:** партнёр криптобиржи BingX. **Полное ТЗ:** `TZ_LeadScout_Bot.md` v1.2 (обязательно к прочтению). **План разработки:** `CURSOR_Guide_LeadScout.md`. Система автоматизирует утверждённую стратегию поиска (карта соответствия — §19 ТЗ).

---

## Что это

LeadScout — мультиагентная система поиска и первичного контакта с лидами для партнёрской программы биржи. Лиды двух типов: **трейдеры** (потенциальные рефералы) и **партнёры** (блогеры/админы с аудиторией). Система мониторит публичные сообщения в соцсетях и форумах на четырёх языках (RU/EN/ES/FR, далее PT/AR), находит людей с «горячими» сигналами (жалобы на комиссии биржи, активная торговля фьючерсами, поиск площадки), квалифицирует их через LLM и готовит персонализированное первое сообщение в одном из четырёх тонов захода (ловля боли / полезный ответ / тёплое интро / партнёрский питч). Целевые страны первой волны: РФ, Киргизия, Перу, Аргентина, Бразилия, Сенегал, Буркина-Фасо, Того, Нигерия, Пакистан (high-risk) — гео-флаги и карточные ограничения зашиты в конфиг. Отправка — с Telegram-аккаунта оператора после его подтверждения одной кнопкой.

## Ключевой принцип

**Human-in-the-loop.** Система никогда не отправляет холодные сообщения сама — только после кнопки Approve от оператора. Rate limits на отправку неотключаемы. Реферальная ссылка не публикуется в открытых сообщениях. Это архитектурные требования, а не настройки.

## Конвейер

```
Scout-агент (ищет новые группы/чаты → карточка оператору → в реестр)
      ↓
Коннекторы (Telegram/Reddit/YouTube/VK/парсер) → поток сообщений в БД
      ↓
Триггер-фильтр (словари по языкам, отсекает ~90–95% без LLM)
      ↓
LLM-скоринг (Claude Haiku, строгий JSON: score 0–100, тип, причины, факты)
      ↓
LLM-черновик первого сообщения (Claude Sonnet, язык лида, персонализация)
      ↓
Карточка лида в управляющий Telegram-бот оператора
      [Отправить] [Редактировать] [Пропустить] [Не лид]
      ↓
Sender (Telethon, очередь, лимиты, ночная тишина, обработка FloodWait)
      ↓
Диалог-менеджер: входящие ответы → черновик реплики → снова Approve
      ↓
CRM: воронка до «зарегистрировался → настроил кэшбэк → пополнил → активно торгует», утренний дайджест, /stats, модуль реактивации (v2)
```

## Агенты (воркеры над общей БД)

1. **Scout** — пополняет реестр источников (каналы, сабреддиты, сайты), оценивает их качество, предлагает оператору на подключение.
2. **Monitor** — коннекторы сбора сообщений (Telethon-стрим + опрос API по расписанию + универсальный HTTP-парсер для сайтов без API).
3. **Scorer** — квалификация кандидатов через Claude API.
4. **Drafter / Replier** — тексты первого касания и реплик в диалогах (база знаний о бирже — отдельный файл, редактируется заказчиком).
5. **Control Bot** — интерфейс оператора в Telegram (aiogram): карточки, кнопки, команды, статистика.

## Режимы

- **M1 Радар** — только карточки лидов, без отправки.
- **M2 Копайлот** (основной) — карточки + отправка по Approve + черновики реплик в диалогах.
- **M3 Автодиалог** (v2) — автоответы только в диалоге, где лид уже ответил и оператор включил тумблер для этого диалога; жёсткие правила эскалации на оператора.

## Стек

Python 3.11+ · Telethon (мониторинг и отправка, два разных аккаунта) · aiogram 3 (управляющий бот) · PRAW · YouTube Data API · VK API · requests/BeautifulSoup + Playwright (парсер) · anthropic SDK (Haiku для скоринга, Sonnet для текстов) · SQLAlchemy + SQLite → Postgres · APScheduler · деплой на VPS (systemd).

## Границы объёма

- Instagram/Facebook/WhatsApp — только ручной режим (оператор присылает ссылку, система делает карточку): программный сбор там закрыт.
- Discord — только официальный Bot API в серверах, куда бота пригласили; self-bot запрещён.
- X — официальный платный API, включается отдельным решением (v2).
- Словари триггеров: первую версию генерирует разработчик через LLM по спецификации ТЗ §7/§18, заказчик валидирует; дальнейшее развитие — командами бота без участия разработчика.
- База фактов о бирже (`knowledge/bingx_facts.md`) и гео-списки — контент заказчика, разработчик делает шаблоны.

## Этапность и критерий приёмки MVP

MVP = Telegram + Reddit мониторинг, фильтр, скоринг, карточки (режим M1). Критерий: за 24 часа ≥ 20 релевантных карточек, доля ложных («Не лид») < 30%. Далее: отправка и диалоги (M2) → остальные коннекторы, Scout, парсер, статистика → v2 (автодиалог, Discord, X, новые языки).
